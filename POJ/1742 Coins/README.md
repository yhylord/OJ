POJ 1742 Coins

【题目大意】

给出N(1 <= N <= 100)种面值的硬币和每种面值硬币的个数，求通过累加能凑出多少种不超过M(1 <= M <= 100000)的总钱数。

【算法分析】

很明显的多重背包问题，用f(i, j)表示到第i种面值钱为j是否可行。我们根据容量模面值的余数分组，每组内就可以从一个连续区间转移过来，若转移队列为空则不可行。

然而很遗憾，这样写会因为常数问题而超时。考虑按照以往的两重循环顺序来求解，记录g(i, j)表示到第i种面值钱为j需要用第i种面值的个数，当不超过限制的个数时才能转移。

事实上，这两种方法本质上是一样的，记录个数其实代表了模分值分组后，同组内最近的可行解离当前值的距离，那么若距离超出限制说明转移队列为空。时间复杂度O(NM)。
