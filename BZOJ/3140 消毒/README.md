BZOJ 3140 消毒

首先题目给出a * b * c <= 5000，令a <= b <= c，易知a <= 17，这是个很好的可利用条件。

我们发现，假如我们付出了k的代价，那我们必然取k * b * c或a * k * c或a * b * k的立方体。

同时，我们发现，付出k的代价，和付出k次1的代价其实是等价的，即我们每次可以取走一个平面。

考虑只在平面上的情况，就相当于每次取走一行或一列求最小次数，这就是经典的二分图最小点覆盖问题了。

那么，我们可以DFS最小的第三维取走哪几层，此后的都必须在二维内取完，用匈牙利算法解决即可。

此外一个明显的剪枝是，答案最大为a，所以在匈牙利算法的过程中发现当前答案已经大于等于a即可跳出。

时间复杂度O(2^a * a^2 * b * c)。
